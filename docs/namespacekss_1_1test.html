void kss::test::skip ( )

Call this within a test if you want to skip it. Typically this would be the first line of your test, but it does not have to be. The test will be aborted at the point this is called, and the test will be marked as skipped regardless of whether it had passed or failed up to that point.

If you place this in the beforeEach method, then all the test cases for that test suite will be skipped. If you place it in the afterEach method, then the test cases for that suite will run, but will still be marked as skipped.

Note that the skip is implemented by throwing an exception and must not be caught or the skip will not be performed. This exception is NOT subclassed from std::exception, so as long as your code does not catch (...) it will be fine.
